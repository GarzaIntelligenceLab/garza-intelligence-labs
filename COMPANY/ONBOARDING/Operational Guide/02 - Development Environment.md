# 02 - Development Environment

> **Status:** Approved
>
> **Version:** 1.0
>
> **Last Updated:** July 2026

---

## Purpose

Establish a consistent development environment for every Garza Intelligence Labs team member.

A standardized environment reduces unnecessary setup time, minimizes configuration issues, and allows every employee to work from the same operational foundation.

---

## Requirements

Before continuing, complete **01 - Required Software.md**.

The following software should already be installed:

- Visual Studio Code
- Git
- GitHub Desktop
- GitHub Account
- Node.js
- Figma
- ChatGPT
- Slack
- Markdown All in One (VS Code Extension)

---

## Initial Setup

### Step 1 — Sign Into GitHub

Sign into your GitHub account through:

- GitHub Desktop
- Visual Studio Code

Verify authentication before continuing.

---

### Step 2 — Open the GIL Workspace

Open the Garza Intelligence Labs workspace in Visual Studio Code.

Always work from the root GIL folder.

Do not create isolated project folders outside the company workspace unless approved.

---

### Step 3 — Verify Folder Structure

Confirm the standard GIL folder structure exists.

The folder hierarchy should remain uniform across all development environments.

Changes to the structure require executive approval.

---

### Step 4 — Verify Extensions

Required VS Code extensions:

- Markdown All in One

Recommended extensions may be added as Garza Intelligence Labs grows.

---

### Step 5 — Configure Git

Verify:

- Git is installed.
- GitHub Desktop detects Git.
- Visual Studio Code detects Git.
- Your GitHub account is authenticated.

---

### Step 6 — Verify Markdown

Create a temporary Markdown document.

Confirm:

- Headings render correctly.
- Lists render correctly.
- Markdown preview functions properly.

Delete the temporary file after verification.

---

### Step 7 — Verify Development Environment

Confirm:

- GitHub authentication is active.
- VS Code recognizes Git.
- Markdown support is functioning.
- Required software opens successfully.

The workstation is now operational.

---

## Development Standards

Every Garza Intelligence Labs workstation should maintain:

- Consistent folder structure
- Consistent software versions whenever practical
- Consistent documentation standards
- Consistent Markdown formatting

Uniformity reduces unnecessary operational complexity.

---

## Verification Checklist

□ Required software installed

□ GitHub authenticated

□ Git configured

□ VS Code configured

□ Markdown functioning

□ Workspace verified

□ Development environment operational
